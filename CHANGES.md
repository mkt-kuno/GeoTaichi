# GeoTaichi Environment Update - Summary

## Overview
This update modernizes GeoTaichi to support the latest Python versions (3.9-3.12) and adds new backend support for broader hardware compatibility.

## Key Changes

### 1. Python Version Support
- **Previous**: Python 3.8 only (with Taichi 1.6.0)
- **Current**: Python 3.9 through 3.12 (with Taichi 1.7.2+)
- **Rationale**: 
  - Taichi 1.7.2+ requires Python 3.9 minimum
  - Python 3.12 is now fully supported
  - Better compatibility with modern development tools

### 2. Dependency Updates
All dependencies updated to support Python 3.9-3.12:
- `taichi>=1.7.2,<1.8.0` (was 1.6.0)
- `shapely>=1.8` (was ==1.8)
- `pynvml>=11.4.1` (was ==11.4.1)
- Added `gmsh` (required for mesh generation)
- Added `numpy` (implicit dependency now explicit)

### 3. New Backend Support: Vulkan
Added Vulkan backend support alongside CPU and CUDA:
```python
from geotaichi import init

# Use Vulkan backend (cross-platform GPU)
init(arch='vulkan', device_memory_GB=4.0)
```

Benefits:
- Cross-platform GPU support (Windows, Linux, macOS)
- Alternative to CUDA for non-NVIDIA GPUs
- Default 4GB memory allocation for Vulkan

### 4. Improved GPU Handling
- Automatic fallback from GPU to CPU when NVIDIA GPU unavailable
- Better error messages for GPU initialization failures
- Lazy imports for GPU-specific libraries (pynvml)

### 5. Package Structure Improvements
- Created proper `pyproject.toml` for pip/uv installation
- Added `third_party` package to distribution
- Made optional imports lazy (gmsh, trimesh.gmsh)
- Fixed package discovery for proper `pip install`

### 6. Updated Configuration Files

#### pyproject.toml (NEW)
Modern Python packaging configuration:
- Defines project metadata
- Specifies Python version constraints
- Lists all dependencies
- Configures package discovery

#### requirements.txt (NEW)
Generated by uv for reproducible builds:
- Pinned versions of all dependencies
- Includes transitive dependencies
- Easy installation: `pip install -r requirements.txt`

#### geotaichi_env.yml (UPDATED)
Conda environment file updated:
- Python 3.11 (recommended version)
- All dependencies updated to compatible versions
- Simplified structure

#### requirements.sh (UPDATED)
Bash installation script updated:
- Updated package versions
- Added gmsh dependency
- Added version constraints

### 7. Backend Architecture Updates
Updated all backend-dependent code to support Vulkan:

**Linear Solvers**:
- `MatrixFreeCG.py`
- `MatrixFreeBICG.py`
- `MatrixFreeBICGSTAB.py`
- `MatrixFreePBICGSTAB.py`

**Utility Functions**:
- `PrefixSum.py`
- `RadixSort.py`

All now recognize Vulkan, Metal, and generic GPU backends as GPU-like architectures.

## Installation Methods

### Method 1: Using pip
```bash
git clone https://github.com/Yihao-Shi/GeoTaichi
cd GeoTaichi
pip install -e .
```

### Method 2: Using uv (Recommended for developers)
```bash
pip install uv
git clone https://github.com/Yihao-Shi/GeoTaichi
cd GeoTaichi
uv pip install -e .
```

### Method 3: Using conda
```bash
git clone https://github.com/Yihao-Shi/GeoTaichi
cd GeoTaichi
conda env create -f geotaichi_env.yml
conda activate geotaichi
```

## Migration Guide

### For Users
If you're using GeoTaichi with Python 3.8:
1. Upgrade to Python 3.9 or higher
2. Reinstall GeoTaichi: `pip install -e .`
3. No code changes required!

### For Developers
If you've been using PYTHONPATH:
1. Remove PYTHONPATH modifications
2. Install as package: `pip install -e .`
3. Import from anywhere: `from geotaichi import init`

## Known Limitations

1. **Vulkan Support**: While the backend initializes, full Vulkan support depends on:
   - System Vulkan drivers
   - GPU Vulkan compatibility
   - Taichi's Vulkan implementation

2. **gmsh Requirement**: Some advanced meshing features require gmsh to be installed separately.

3. **GPU Memory Detection**: Vulkan backend uses fixed 4GB default (no dynamic detection yet).
